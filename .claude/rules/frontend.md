# Frontend conventions (Vite / React)

Applies when editing code under `frontend/src/`.

- **Stack**: Vite 5 + React 18 + Tailwind. Page-level features live in
  `src/features/`, shared API clients in `src/shared/`, reusable UI in
  `src/components/`. Routes are lazy-loaded in `App.jsx`.
- **Vendored design system**: `src/vendor/` holds vendored Mainspring tokens/UI —
  don't hand-edit vendored files; they're regenerated by `scripts/vendor_mainspring_*`.
- **File-size cap**: `npm run check:architecture` enforces a per-file LOC ceiling
  (~2600). Split large components rather than letting a file balloon.
- **API calls** go through the clients in `src/shared/` (axios); in dev, `/api` is
  proxied to the backend on `:8000`.
- **Tests**: colocated `*.test.jsx` using Vitest + Testing Library (jsdom). Run with
  `npm test` from `frontend/`; typecheck with `npm run typecheck`.
- **Before pushing**: `npm run build` must pass — it's a CI gate and catches import
  and type errors.
- **Secrets**: only `VITE_*` env vars are exposed to the client; never put real
  secrets in frontend env.
