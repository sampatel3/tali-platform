Metric surfaces — one consistent system

Today there are six metric surfaces across Home and Job pages, each inventing its own labels, ordering, number formatting and colour rules for what are often the same underlying numbers. This proposes one shared vocabulary, one tile component, and one formatting standard.

5 · Formatting standard

LabelsUPPERCASE mono, --mute, .1em tracking. Title-case in source, CSS uppercases. One label per concept.
CountsThousands separators ≥ 1,000 → 52,594. Geist display, 26px (tile) / 22px (funnel).
MoneyWhole dollars, always $spent / $cap + bar. No cap → — · no cap. Never cents.
PercentInteger + %. Used in sub-line, never as the primary budget value.
Sub-lineAlways present, one line, lowercase, ≤ ~22 chars. Context or the single most useful secondary number.
Colour — 3 meanings only
needs you — Review, Awaiting you, over-budget
volume — neutral counts
terminal / context — Rejected, sub-lines
Retire ad-hoc red/green dot grid → purple / mute.

One decision needs your call: "Awaiting you" = pending agent decisions (the thing you action), while the Review funnel stage stays in the funnel only. Today the Jobs header's "review queue" actually shows the Review stage count — this proposal repoints it to pending decisions so the label means the same thing on every surface. Shout if you'd rather keep it as the stage count.
