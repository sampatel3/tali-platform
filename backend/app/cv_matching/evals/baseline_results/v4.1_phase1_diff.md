## How to populate this diff

```bash
# Prerequisites:
#   - ANTHROPIC_API_KEY in env or platform settings
#   - anthropic SDK >= 0.39 if you want tasks 1.4 (tool-use) / 1.5
#     (prompt caching) to also fire — neither is required for 1.9
cd backend

# 1. Run v3 baseline (skip if a recent v3 snapshot already exists in
#    baseline_results/).
python -m app.cv_matching.evals.run_evals --version v3 --no-cache

# 2. Run v4.1 baseline. Force --no-cache so the v4.1 path is exercised
#    end-to-end against fresh Claude calls.
python -m app.cv_matching.evals.run_evals --version v4.1 --no-cache

# 3. Run them head-to-head and capture the per-case comparison.
python -m app.cv_matching.evals.run_evals --version both --no-cache \
    > v4.1_phase1_run.txt
```

The harness writes one snapshot per version into this directory:
`cv_match_v3.0_<ts>.json` and `cv_match_v4.1_<ts>.json`. Use those as
the input to the metrics described below.
