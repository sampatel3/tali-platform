# v4.2 vs v4.1 / v3 — Phase 2 baseline diff

## Status

**Snapshot pending live run.** Phase 2 code is in place; populating
the diff requires running the eval harness against real Anthropic
calls (not stubbed) and a configured Voyage embedding key (or the
`mock` provider for embedding-light validation).

## How to populate this diff

```bash
cd backend

# 1. v4.2 baseline (with archetype routing). Set EMBEDDING_PROVIDER=voyage
#    and a real VOYAGE_API_KEY for production-shape numbers; mock provider
#    will still produce valid outputs but with degenerate cosine routing.
python -m app.cv_matching.evals.run_evals --version v4.2 --no-cache

# 2. Head-to-head against v4.1 and v3.
python -m app.cv_matching.evals.run_evals --version both --no-cache  # v3 + v4.1

# 3. Run v4.2 alone on the same cases and diff against v3 manually
#    (the harness only does pairwise v3+v4.1 today; extend to v4.2
#    when the archetype rubric library is fleshed out).
```
