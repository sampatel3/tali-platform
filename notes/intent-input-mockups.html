Four ways to capture role intent

All four mockups replace the same single textarea. Each handles must-have / preferred / nice-to-have / constraints differently. Open each, click around — they're interactive enough to feel.

01

Prose first, agent extracts chips

Recommended

Default role intent

Write naturally. The agent extracts what matters and shows you how it'll read this — review and correct before saving.

~6s to extract · uses ~$0.01 of your role budget

Strengths

02

Slash-command inline blocks

Linear / Notion feel

Default role intent

Type freely. Use / to add a typed block (must, preferred, nice, constraint). Plain prose still works — only tag what matters.

No "10x engineer" framing, no PhD gating. We need owners.
● Must-have /m ● Preferred /p ● Nice-to-have /n ● Constraint /c
Strengths
Single source of truth — what you see is what the agent gets. No extraction cost, no stale state. Recruiters who want pure prose can still write pure prose.
Risks
The slash shortcut needs discoverability — first-time users may miss it. Onboarding has to teach the pattern, otherwise it degrades to a plain textarea.

    Strengths
    Zero ambiguity. Every criterion has a category. Easy to scan, easy to delete, easy to reorder. Best if recruiters already think in checklists.
    Risks
    Contradicts your microcopy ("not a checklist of rules"). High friction — forces categorization before the recruiter has thought it through. Loses nuance like "they probably get it if they've worked at Stripe."
